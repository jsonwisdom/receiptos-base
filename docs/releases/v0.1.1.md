# @jsonwisdom/farcaster-verify v0.1.1

Type-safety release for the authority=false Farcaster Mini App manifest verifier.

## Summary

v0.1.1 adds TypeScript declarations so downstream projects get compile-time guardrails for the same doctrine enforced by CI.

This release keeps the verifier focused on replayable evidence:

```text
authority=false
RFC8785 canonicalization
Replay First. Verify Everything.
```

## Added

- `index.d.ts` TypeScript declarations
- `types` field in `package.json`
- typed Farcaster manifest surface
- typed manifest lock surface
- literal doctrine types for lock invariants

## Type surface

```ts
FarcasterPermission
FarcasterDeveloper
FarcasterMiniAppManifest
FarcasterManifestLock
Doctrine
CommandMap
PackageExports
```

## Compile-time doctrine

The following constraints are now represented in TypeScript:

```ts
authority: false
canonicalization: "RFC8785"
canonical_source: "manifest.jcs"
status: "LOCKED"
```

This means consumers get editor and `tsc` feedback before CI runs.

## Why it matters

Before v0.1.1, invalid manifests were rejected by the verifier and CI.

With v0.1.1, TypeScript consumers can catch doctrine violations earlier:

- `authority: true` fails type checking
- non-RFC8785 canonicalization fails type checking
- wrong canonical source fails type checking
- unsupported lock status fails type checking

## Existing verification stack

This release preserves the full v0.1.0 verifier stack:

- RFC8785/JCS canonicalization via `json-canonicalize`
- semantic verifier
- manifest lock verifier
- valid RFC8785 vectors
- invalid rejection vectors
- deterministic fuzz mutation vectors
- reusable GitHub Actions workflow
- lock rotation runbook
- transparency log runbook
