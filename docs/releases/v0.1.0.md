# @jsonwisdom/farcaster-verify v0.1.0

Initial release of the authority=false Farcaster Mini App manifest verifier.

## What this release provides

`@jsonwisdom/farcaster-verify` turns Farcaster Mini App manifest verification into a replayable CI gate:

```text
manifest.json -> RFC8785 canonicalization -> manifest.jcs -> sha256 -> manifest.lock.json -> CI receipt
```

## Included checks

- RFC8785/JCS canonicalization using `json-canonicalize`
- semantic manifest validation
- HTTPS-only URL doctrine
- Farcaster permission whitelist
- icon and screenshot constraints
- manifest-to-canonical drift detection
- manifest lock SHA-256 verification
- `authority=false` enforcement
- future-dated lock rejection
- valid RFC8785 vectors
- invalid rejection vectors
- deterministic fuzz mutation vectors

## CLI commands

```bash
farcaster-verify-rfc8785
farcaster-verify-invalid
farcaster-verify-fuzz
farcaster-verify-semantic
farcaster-verify-lock
```

## Local verification

```bash
npm install
npm run verify:farcaster
```

## Consumer usage

```bash
npm install -D @jsonwisdom/farcaster-verify
npx farcaster-verify-rfc8785
npx farcaster-verify-invalid
npx farcaster-verify-fuzz
npx farcaster-verify-semantic
npx farcaster-verify-lock
```

## Doctrine

This package records replayable evidence. It does not assert authority.

```text
authority=false
Replay First. Verify Everything.
```
