## Verification Model

ReceiptOS v0.2.0 verifies evidence through this sequence:

1. Load `evidence/anchors.json`.
2. Replay each anchor independently.
3. Cross-check canonical receipt hashes.
4. Verify on-chain witness data where configured.
5. Monitor revocation status for revocable attestations.
6. Fail closed on mismatch, drift, missing receipt, or revoked anchor.

## Philosophy

Replay first. Authority is false. Every production claim must trace to deterministic local receipts, Git commits, on-chain attestations, or anchored metadata.

## KARENN11 Status

KARENN11 remains the reference production artifact for the ReceiptOS witness architecture. The v0.2 foundation moves the project from a single-witness pipeline to a manifest-driven provenance engine.

## How to Verify

```bash
git checkout receiptos-v0.2.0
bash scripts/verify-all-anchors.sh evidence/anchors.json
```

If `BASE_RPC_URL` is configured, the verifier also checks the configured on-chain witness and revocation state.
